# M365 Copilot Chat API 接入说明（含 Tenant ID / Client ID / Client Secret 获取）

更新时间：2026-03-12


## 1. 接口形态与核心结论

Microsoft 365 Copilot Chat API（Graph beta）不是单一 `/chat` 端点，而是两步：

1. `POST /beta/copilot/conversations` 创建会话，拿到 `conversationId`
2. `POST /beta/copilot/conversations/{conversationId}/chat` 或 `/chatOverStream` 继续对话

该 API 仍为 beta，官方说明不建议用于生产环境，且要求调用用户具备 Copilot 许可。


## 2. 前提条件（必须满足）

1. 账号与许可
   - 工作或学校账号（不支持个人 Microsoft 账号）
   - 用户拥有 Microsoft 365 Copilot 许可（并具备对应基础订阅）

2. 权限类型
   - 仅支持 Delegated（委派权限）
   - 不支持 Application（应用权限）

3. Microsoft Graph 委派权限（缺一不可）
   - `Sites.Read.All`
   - `Mail.Read`
   - `People.Read.All`
   - `OnlineMeetingTranscript.Read.All`
   - `Chat.Read`
   - `ChannelMessage.Read.All`
   - `ExternalItem.Read.All`


## 3. 如何获取 Application (client) ID、Directory (tenant) ID、Client Secret

下面按微软官方门户实际操作路径写，适合首次配置。

### 3.1 进入正确租户

1. 打开 `https://entra.microsoft.com`
2. 登录后，右上角切换到你要接入的目标租户
3. 进入 `Identity` -> `Applications` -> `App registrations`

如果你在多个租户中操作，先确认右上角目录名称是否正确，再创建应用。

### 3.2 新建应用注册（若已有可跳过）

1. 在 `App registrations` 页面点击 `New registration`
2. 填写：
   - `Name`：例如 `m3652api-local`
   - `Supported account types`：建议选择 `Accounts in this organizational directory only`
3. 点击 `Register`

完成后会进入该应用的 `Overview` 页面。

### 3.3 获取 Directory (tenant) ID 与 Application (client) ID

在应用 `Overview` 页面可直接看到：

- `Application (client) ID`：这就是 `client_id`
- `Directory (tenant) ID`：这就是 `tenant_id`

可直接复制使用。

### 3.4 创建 Client Secret（仅在需要机密客户端时）

1. 进入应用左侧 `Certificates & secrets`
2. 在 `Client secrets` 标签页点击 `New client secret`
3. 填写：
   - `Description`：例如 `m3652api-dev-secret`
   - `Expires`：建议按公司安全策略选择最短可接受周期
4. 点击 `Add`
5. 立刻复制 `Value` 列的密钥值并安全保存

重要：

- 你离开页面后无法再次看到完整 secret 值，只能重新新建
- 建议把 secret 放入密码管理器或 Key Vault，不要写入代码仓库

### 3.5 给应用添加 Graph 委派权限并管理员同意

1. 进入 `API permissions` -> `Add a permission`
2. 选择 `Microsoft Graph` -> `Delegated permissions`
3. 添加第 2 节列出的全部权限
4. 回到 `API permissions` 页面点击 `Grant admin consent for <TenantName>`
5. 确认每个权限状态为已授予（Granted）

### 3.6 配置 Redirect URI（授权码登录必需）

由于 Copilot Chat API 只支持 **Delegated**，本项目需要走 **OAuth 2.0 授权码（Authorization Code）** 获取 `refresh_token`，以便服务端后续自动刷新访问令牌。

你需要在 Entra ID 应用中配置 Redirect URI：

1. 进入应用左侧 `Authentication`
2. 点击 `Add a platform`（添加平台）
3. 选择 `Web`
4. 在 `Redirect URIs` 中添加（按你的端口调整）：
   - `http://localhost:8217/m365/oauth/callback`
5. 保存（Save）

说明：

- 这是“本机单用户代理”形态下的推荐配置
- 如果你修改了 `config.yaml` 的端口或希望自定义回调地址，请同时修改 Redirect URI 与配置中的 `m365.redirect-uri`


## 4. 本项目（m3652api）实际需要哪些参数

本仓库当前实现改为静态凭据模式：服务启动时从 `config.yaml` 读取 `m365` 段。

最小必填如下：

- `m365.tenant-id`
- `m365.client-id`
- `m365.client-secret`
- `m365.timezone`（建议显式配置，示例：`Pacific/Auckland`）
- `m365.redirect-uri`（建议显式配置并与 Entra Redirect URI 保持一致）

示例：

```yaml
m365:
  auth-id: "m365-static"
  tenant-id: "<TENANT_ID>"
  client-id: "<CLIENT_ID>"
  client-secret: "<CLIENT_SECRET>"
  timezone: "Pacific/Auckland"
  # App-only scopes（仅作为兜底；Copilot Chat API 仍要求 delegated token）
  scopes:
    - "https://graph.microsoft.com/.default"
  # Delegated scopes（授权码登录使用，用于获取 refresh_token）
  delegated-scopes:
    - "openid"
    - "profile"
    - "offline_access"
    - "https://graph.microsoft.com/User.Read"
    - "https://graph.microsoft.com/Sites.Read.All"
    - "https://graph.microsoft.com/Mail.Read"
    - "https://graph.microsoft.com/People.Read.All"
    - "https://graph.microsoft.com/OnlineMeetingTranscript.Read.All"
    - "https://graph.microsoft.com/Chat.Read"
    - "https://graph.microsoft.com/ChannelMessage.Read.All"
    - "https://graph.microsoft.com/ExternalItem.Read.All"
  redirect-uri: "http://localhost:8217/m365/oauth/callback"
  ocr-langs: "eng"
```

### 4.1 首次授权（浏览器登录，不在命令行交互）

启动服务后，用浏览器打开：

- `http://localhost:8217/m365/oauth/start`

完成登录后，可查看：

- `http://localhost:8217/m365/oauth/status`

当 `has_refresh_token=true` 时，说明服务端已经保存了可用于持续刷新的委派凭据。

### 4.2 上游连通性自检（可选）

如果你想在不写任何 curl 的情况下快速确认：

1. 委派 token 是否可用（`GET /v1.0/me`）
2. Copilot Chat API 是否可创建会话（`POST /beta/copilot/conversations`）

可以在浏览器打开：

- `http://localhost:8217/m365/upstream/check`

说明：

- 该接口会创建一个 Copilot conversation（用于验证权限与许可），不会发送聊天内容
- 如只想验证委派 token，可访问：`http://localhost:8217/m365/upstream/check?copilot=0`


## 5. 请求调用最小示例

### 5.1 创建会话

```bash
curl -X POST "https://graph.microsoft.com/beta/copilot/conversations" \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d "{}"
```

### 5.2 继续聊天（同步）

```bash
curl -X POST "https://graph.microsoft.com/beta/copilot/conversations/<CONVERSATION_ID>/chat" \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "message": { "text": "Summarize my latest unread emails." },
    "locationHint": { "timeZone": "Pacific/Auckland" }
  }'
```

### 5.3 继续聊天（流式 SSE）

```bash
curl -N -X POST "https://graph.microsoft.com/beta/copilot/conversations/<CONVERSATION_ID>/chatOverStream" \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{
    "message": { "text": "Give me a concise action plan." },
    "locationHint": { "timeZone": "Pacific/Auckland" }
  }'
```


## 6. 常见问题排查

1. 找不到 `tenant_id` 或 `client_id`
   - 你不在应用 `Overview` 页面，或租户切错了

2. `client secret` 创建后复制不到
   - 只会显示一次，必须创建后立即复制；丢失只能重建

3. 调用报权限不足（403）
   - 通常是权限没全加，或者没点击 `Grant admin consent`

4. 调用报不支持权限类型
   - 你用了 application token；该 API 只支持 delegated

5. 时间相关回答不准确
   - 缺少 `locationHint.timeZone`，或时区不是 IANA 格式
